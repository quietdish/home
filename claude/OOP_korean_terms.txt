【5가지 핵심 개념】

1️⃣ Class & Object
   → 클래스와 객체
   → "틀을 만들고, 그 틀로 실제 것을 만든다"

2️⃣ Encapsulation
   → 캡슐화
   → "중요한 데이터는 보호하고, 규칙을 정해서만 변경한다"

3️⃣ Inheritance
   → 상속
   → "공통 부분은 부모에서, 고유한 부분은 자식에서"

4️⃣ Polymorphism
   → 다형성
   → "같은 메서드인데 객체에 따라 다르게 동작한다"

5️⃣ Abstraction
   → 추상화
   → "반드시 구현해야 할 것을 정해서 강제한다"

【실제 코드에서의 사용】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

// 클래스 정의
class 학생 {                    // 클래스
    private String 이름;        // 비공개 속성 (캡슐화)
    
    public 학생(String 이름) {   // 생성자
        this.이름 = 이름;
    }
    
    public String 이름_가져오기() { // 게터 메서드
        return 이름;
    }
}

// 객체 생성
학생 학생1 = new 학생("김철수");  // 학생1은 "인스턴스"

// 상속
class 대학생 extends 학생 {      // 대학생은 학생을 상속받음
    @Override
    public void 공부하기() {      // 오버라이드
        System.out.println("대학 과제를 하고 있습니다");
    }
}

// 추상 클래스
abstract class 사람 {           // 추상 클래스
    abstract public void 역할수행();  // 추상 메서드 (구현 안 됨)
}

class 학생2 extends 사람 {        // 반드시 역할수행() 구현해야 함
    @Override
    public void 역할수행() {       // 구현
        System.out.println("공부합니다");
    }
}

   └ "데이터를 private으로 보호하고, 게터/세터로 관리한다"

👨‍👩‍👧 3단계: 상속
   └ "공통 부분을 부모에서 한 번만 정의한다"

   └ "같은 이름의 메서드가 객체에 따라 다르게 동작한다"

🎯 5단계: 추상화
   └ "자식이 반드시 구현해야 할 메서드를 정한다"
